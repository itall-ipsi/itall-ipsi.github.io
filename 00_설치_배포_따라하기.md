# 잇올 입시평가 — 설치·배포 따라하기 가이드

이 시스템은 **두 부분**으로 나뉩니다. 각각 따로 올립니다.

| 부분 | 무엇 | 어디에 | 파일 |
|---|---|---|---|
| ① 채점 | 답을 채점·기록 | **구글 Apps Script** | `채점시스템_GAS/` 안의 Code.gs·Data.gs·Index.html·Admin.html |
| ② 화면(허브) | 요약·상세·테스트를 한 화면에 | **웹 호스팅**(넷리파이/깃허브) | `index.html` + `white/` 폴더 + 요약·세부·해설·부록 HTML |

> 핵심 원칙: **`채점시스템_GAS` 폴더의 파일은 호스팅에 올리지 않습니다.** (Apps Script 안에서만 작동) 호스팅에는 화면(허브) 파일만 올립니다.

---

# PART A. 채점 시스템 업데이트 (Apps Script 재배포)

이미 한 번 배포해 두셨으니, **내용만 새 버전으로 바꾸고 다시 배포**하면 됩니다. URL은 그대로 유지됩니다.

### A-1. Apps Script 편집기 열기
1. 결과가 쌓이는 **구글 시트**를 엽니다.
2. 상단 메뉴 **확장 프로그램 → Apps Script** 클릭.

### A-2. 파일 4개 내용 교체
편집기 왼쪽에 파일 목록(Code.gs, Data.gs, Index, Admin)이 있습니다. **각 파일마다** 아래를 반복:
1. 해당 파일 클릭.
2. 코드 영역 안을 클릭 → **Ctrl+A**(전체 선택) → **Delete**(전부 지움).
3. 이 폴더 `채점시스템_GAS/`의 **같은 이름 파일 내용을 복사**해 붙여넣기(**Ctrl+V**).

교체할 파일: **Code.gs · Data.gs · Index(html) · Admin(html)** 네 개.
- `Code.gs` : 재응시 허용 + 이름 기반 집계로 바뀐 새 버전
- `Data.gs` : 그동안 수정한 문항·정답이 반영된 최신본
- `Index` : 응시 화면(이름 필수·재응시 배너)
- `Admin` : 대시보드(공식/재응시 구분)
> `appsscript.json`은 바뀐 게 없어 그대로 둬도 됩니다.

4. 상단 **💾 저장** (Ctrl+S).

### A-3. 새 버전으로 재배포  ← 가장 중요
**저장만으로는 적용되지 않습니다.** 반드시 새 버전 배포를 해야 `/exec` 주소에 반영됩니다.
1. 우측 상단 **배포 → 배포 관리**.
2. 기존 배포 항목의 **연필(✏️ 편집)** 클릭.
3. **버전** 드롭다운 → **새 버전** 선택.
4. **배포** 클릭. (권한 화면이 다시 뜨면 본인 계정으로 **허용**)
5. 끝. **웹 앱 URL은 그대로** 유지됩니다.

### A-4. 동작 확인
- 응시: `웹앱URL/exec` 열기 → 평가 선택 → **이름/소속 입력** → 제출 → 점수 바로 표시.
- 같은 이름으로 다시 풀면 상단에 **"재응시 — 공식 집계 미반영"** 배너가 떠야 정상.
- 관리자: `웹앱URL/exec?page=admin` (ADMIN_EMAILS 계정으로 로그인).

> (선택) 과거 테스트 데이터를 깔끔히 지우려면 구글 시트 「응답」 탭에서 2행 이하를 삭제하세요. 헤더(1행)는 두세요.

# PART B. 화면(허브) 웹 호스팅

가장 쉬운 **넷리파이 드롭** 방법을 먼저, 그다음 **깃허브 페이지** 방법을 적었습니다. 둘 중 하나만 하면 됩니다.

### 올릴 파일 / 올리지 말 파일
- ✅ 올림: `index.html`, **`white/` 폴더 통째로**, `부록_정시수능100_저학생부_검정고시길.html`, `00_전체커리큘럼_종합안내.html`, `00_로드맵_12주_커리큘럼.html`
  - (원본 요약·세부·해설·테스트지 등 나머지 HTML도 함께 올려도 무방합니다. 헷갈리면 **`채점시스템_GAS` 폴더만 빼고 전부** 올리세요.)
- ❌ 올리지 않음: **`채점시스템_GAS/` 폴더** (이건 Apps Script 전용)

---

## 방법 1) 넷리파이 드롭 — 가장 쉬움(계정 없이 즉시)
1. 브라우저에서 **app.netlify.com/drop** 접속.
2. 파일 탐색기에서 이 폴더를 열고 **`채점시스템_GAS` 폴더를 잠시 다른 곳으로 빼두기**(올리면 안 되므로). 또는 올릴 파일들만 따로 한 폴더에 모으기.
3. 올릴 파일 전체(= `index.html` + `white` 폴더 + 나머지 HTML)를 **마우스로 한꺼번에 선택해 페이지로 끌어다 놓기(drag & drop)**.
   - `white` 폴더도 함께 끌어다 놓으면 하위 구조가 유지됩니다.
4. 잠시 기다리면 **임의 주소**(예: `https://random-name-123.netlify.app`)가 생성됩니다. → 이게 허브 주소.
5. (선택) 무료 가입하면 그 주소가 유지됩니다. 가입 안 하면 임시 주소입니다.

## 방법 2) 깃허브 페이지
1. **github.com** 로그인(없으면 회원가입).
2. 우상단 **＋ → New repository** → 이름 입력(예: `itall-ipsi`) → **Public** → **Create repository**.
3. 새 저장소 화면에서 **uploading an existing file** 링크 클릭.
4. 올릴 파일들(= `index.html` + `white` 폴더 + 나머지 HTML, **`채점시스템_GAS` 제외**)을 **끌어다 놓기**.
   - `white` 폴더째 끌어다 놓으면 됩니다.
5. 아래 **Commit changes** 클릭.
6. 상단 **Settings → 좌측 Pages** → **Source: Deploy from a branch** → **Branch: main / (root)** → **Save**.
7. 1~2분 뒤 페이지가 켜집니다. 주소: `https://<아이디>.github.io/<저장소이름>/`
8. 그 주소를 열면 허브가 나옵니다.

> 한글 파일명도 정상 작동합니다(자동 인코딩). 허브는 옆 파일을 파일명으로 부르므로, **index.html과 white 폴더가 같은 위치**에 있어야 합니다.

# 자주 나는 문제
- **"google is not defined"**: GAS의 Index/Admin을 `/exec`가 아니라 **파일로 직접 열었을 때** 납니다. 반드시 **웹앱 URL**로 접속하세요(관리자는 `?page=admin`). 이 파일들은 **호스팅에 올리지 마세요**.
- **요약/상세가 아직 크림색**: 허브가 `white/` 폴더를 못 찾는 경우입니다. `index.html`과 `white` 폴더가 **같은 위치**에 올라갔는지 확인.
- **"공식 제출 1회 제한 / 지난 제출…" 옛 화면이 뜸**: 아직 **옛 GAS가 배포 중**입니다. PART A-3(새 버전 배포)을 다시 하세요.
- **다른 사람이 못 푸는 문제**: 새 버전에서는 **이름이 다르면** 각자 공식 응시가 됩니다. 같은 이름으로만 막히는(재응시) 것이 정상입니다.
